- **Java:** 25 remains required for this project and the 26.2 line.

## Compatibility Metadata Policy

- Gradle compile inputs stay pinned to exact artifact versions for this build.
- Loader metadata should advertise an open Minecraft floor for this release line, not a closed tested range.
- Fabric should use a minimum floor such as `>=26.2`.
- NeoForge should use an open-ended range such as `[26.2,)`.
- This does not claim later Minecraft versions are tested; it only avoids blocking compatible patch/minor runtimes by metadata alone.
- **Policy risk:** this open floor may allow a future incompatible Minecraft runtime to attempt loading the mod. That is intentional for this release's compatibility policy unless a verified hard incompatibility is found.

## 26.2 Target Recheck

### CarvedPumpkinBlock

- `net.minecraft.world.level.block.CarvedPumpkinBlock#trySpawnGolem(Level, BlockPos)` remains private with descriptor `(Lnet/minecraft/world/level/Level;Lnet/minecraft/core/BlockPos;)V`.
- `clearPatternBlocks` and `updatePatternBlocks` remain public static helpers.
- No retarget needed for `CarvedPumpkinBlockMixin` or `GolemCreationHandler` based on descriptor evidence.

### IronGolem, LivingEntity, Mob, and Villager

- `IronGolem` remains `net.minecraft.world.entity.animal.golem.IronGolem`.
- `IronGolem#doHurtTarget(ServerLevel, Entity)`, `hurtServer(ServerLevel, DamageSource, float)`, `mobInteract(Player, InteractionHand)`, and `setPlayerCreated(boolean)` remain present.
- `LivingEntity#setLastHurtByMob(LivingEntity)` remains declared on `LivingEntity`, descriptor `(Lnet/minecraft/world/entity/LivingEntity;)V`.
- `Mob#setTarget(LivingEntity)` remains declared on `Mob`, descriptor `(Lnet/minecraft/world/entity/LivingEntity;)V`.
- `Villager#spawnGolemIfNeeded(ServerLevel, long, int)` remains present, descriptor `(Lnet/minecraft/server/level/ServerLevel;JI)V`.

### SpawnEggItem

- `SpawnEggItem#useOn(UseOnContext)` remains descriptor `(Lnet/minecraft/world/item/context/UseOnContext;)Lnet/minecraft/world/InteractionResult;`.
- The private normal-spawn helper changed to:
  `spawnMob(EntityType<?>, LivingEntity, ItemStack, ServerLevel, BlockPos, boolean, boolean)`.

  `(Lnet/minecraft/world/entity/EntityType;Lnet/minecraft/world/entity/LivingEntity;Lnet/minecraft/world/item/ItemStack;Lnet/minecraft/server/level/ServerLevel;Lnet/minecraft/core/BlockPos;ZZ)Lnet/minecraft/world/InteractionResult;`
- The helper still invokes:
  `EntityType.spawn(ServerLevel, ItemStack, LivingEntity, BlockPos, EntitySpawnReason, boolean, boolean)`.
- Exact redirect method descriptor to use:
  `spawnMob(Lnet/minecraft/world/entity/EntityType;Lnet/minecraft/world/entity/LivingEntity;Lnet/minecraft/world/item/ItemStack;Lnet/minecraft/server/level/ServerLevel;Lnet/minecraft/core/BlockPos;ZZ)Lnet/minecraft/world/InteractionResult;`
- Exact invoke target descriptor remains:
  `Lnet/minecraft/world/entity/EntityType;spawn(Lnet/minecraft/server/level/ServerLevel;Lnet/minecraft/world/item/ItemStack;Lnet/minecraft/world/entity/LivingEntity;Lnet/minecraft/core/BlockPos;Lnet/minecraft/world/entity/EntitySpawnReason;ZZ)Lnet/minecraft/world/entity/Entity;`
- **Required retarget:** update `SpawnEggItemMixin` redirect method descriptor to include the leading `EntityType` and `ServerLevel`.

### BaseSpawner

- `BaseSpawner#serverTick(ServerLevel, BlockPos)` remains descriptor `(Lnet/minecraft/server/level/ServerLevel;Lnet/minecraft/core/BlockPos;)V`.
- The server spawn path still calls `TagValueInput.create(...)`, `EntityType.loadEntityRecursive(ValueInput, Level, EntitySpawnReason, EntityProcessor)`, and `ServerLevel.tryAddFreshEntityWithPassengers(Entity)`.
- The display-preview path changed. `getOrCreateDisplayEntity(Level, BlockPos)` now calls:

- Keep the server-spawn redirect target on the `ValueInput` plus `EntitySpawnReason` overload:
  `Lnet/minecraft/world/entity/EntityType;loadEntityRecursive(Lnet/minecraft/world/level/storage/ValueInput;Lnet/minecraft/world/level/Level;Lnet/minecraft/world/entity/EntitySpawnReason;Lnet/minecraft/world/entity/EntityProcessor;)Lnet/minecraft/world/entity/Entity;`
- **Required retarget:** update `BaseSpawnerMixin` preview redirect target and parameter type from `EntitySpawnReason` to `EntitySpawnRequest`.

### Client Render State

## Redstone Golem 26.2 Spike

- Rejected hook: no new entity type, block, or loader-specific creation path is needed.
- Proceed: yes.

### Redstone Dust healing

- Inspected `src/common/java/dev/charles/multigolem/mixin/IronGolemMixin.java`, `src/common/java/dev/charles/multigolem/interaction/IronGolemHealInteraction.java`, and `src/common/java/dev/charles/multigolem/ability/GolemCombatRules.java`.

- Rejected hook: no vanilla fallback hook or loader event is needed; the existing wrong-item/pass and global-healing cancellation behavior is already common.
- Proceed: yes.

### Threshold detection after damage

- Inspected `src/fabric/java/dev/charles/multigolem/fabric/ability/FabricAbilityEvents.java`, `src/neoforge/java/dev/charles/multigolem/neoforge/ability/NeoForgeAbilityEvents.java`, and existing tick abilities in `src/common/java/dev/charles/multigolem/ability`.

- Rejected hook: loader damage events are not needed for the threshold edge because the current health can be evaluated after damage on the next server tick, preserving Fabric/NeoForge parity.
- Proceed: yes.

### Death overload hook

- Inspected live source for existing `die`, death, and loader death-event hooks and found no current death hook.
- Chosen hook: add a narrow common mixin on `IronGolem#die(DamageSource)` to call `RedstoneAbility.onDeath(ServerLevel, IronGolem)` once for Redstone Golems on the server, then list the new mixin in both Fabric and NeoForge mixin JSON resources.
- Rejected hook: avoid one-loader-only death events; Fabric and NeoForge currently share common mixins for IronGolem behavior.
- Proceed: yes, with compile verification of the `die(DamageSource)` descriptor from the existing 26.2 target note.
